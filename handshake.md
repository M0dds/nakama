## Data-Layer (TanStack Query)

Jede Feature-Area ist `src/lib/queries/<area>.ts` mit derselben Struktur: typed Query-Keys + `*Options(...)`-Reads + Mutation-Funktionen. **Signaturen stehen im Source** (kurz reinschauen statt hier duplizieren) ‚Äî hier nur die nicht-offensichtlichen Konventionen + eine Datei-Landkarte. Durable Fallen (Sync-Lanes, Cover-URLs, Bulk-Upsert, Discriminated-Union, Multi-Season) leben in **¬ßGotchas**.

**Query-Key-Konventionen (entscheidend f√ºrs Invalidieren):**

- `lists.ts` keys sind nach `short_code` indexiert (URL-stabil), **nicht UUID**; Mutations filtern auf `lists.id`. Cross-cutting Writes invalidieren den `["list"]`-Prefix (deckt alle offenen shortCodes).
- `episodes.ts`: konkrete Keys sind `[...episodesQueryKey, limit]` + die Sync-Lane ‚Üí Prefix-Invalidierung clear-t alle Paginations + beide Lanes auf einmal.
- `sharing.ts` keys sind **Prefixe** (`["list-members", listId]`, `["co-watchers", itemId]` ‚Ä¶) damit Realtime ohne Mount-Zeit-id invalidieren kann.

**Datei-Landkarte:**

- `lists.ts` ‚Äî Listen-CRUD, Pin/Reorder, per-row Item-Mutations (`removeListItem`/`moveListItem`), **‚ÄûNeue Folge"-Badge-Engine** (`findItemsWithNewEpisodes`, 14-Tage-Fenster; anime/series ‚Üí folgen, manga ‚Üí kapitel; `ListSummary.newCounts` pro Liste, `ListEntry.hasNewEpisode` pro Item).
- `items.ts` ‚Äî Item by natural key `(type, slug)` (Items effektiv public). `addItemToList` upsertet `items(source,source_id)` ‚Üí Trigger setzt slug ‚Üí `list_items`; liest `result.source` (nicht mehr hardcoded "anilist"); 23505 = schon drin ‚Üí success.
- `episodes.ts` ‚Äî Episoden-Read (resolve via `(type,slug)` + 12 h Stale-Gate + lazy fetch). Writes via `set_episode_watch` / `mark_episodes_watched_upto` / `reset_progress` ‚Äî **alle lane-branchend** server-side (¬ßGotchas ‚Üí Sync-Instanzen). ## Realtime

`src/lib/realtime.ts` exportiert `useRealtimeInvalidation(channelKey, [{table, invalidates}])`. Im Component-Mount wird ein Supabase-Channel aufgemacht, jeder postgres_changes-Event invalidiert die deklarierten Query-Keys. RLS scoped Events server-side, kein Client-Filter n√∂tig.

**Verwendet in:**

- `/` home ‚Üí channel `home`, listens to `episode_watches/episodes/list_items/list_members/list_ownership_transfers`, invalidates `homeQueryKey`-Prefix (transfers landen so live im Logbuch-Feed)
- `/lists` overview ‚Üí channel `lists-overview`, listens to `lists/list_members/list_items/episodes/episode_watches/list_invitations`, invalidates `listsQueryKey` (+ `["invitations","mine"]` f√ºr die Inbox-Karten)
- `/lists/:shortCode` ‚Üí channel `list-{shortCode}`, invalidates `listQueryKey(shortCode) + listsQueryKey + listItemsQueryKey(shortCode)`; Phase 7: `list_invitations`/`list_members` invalidieren auch die `["list-members"]`/`["list-invitations"]`-Prefixe (Roster + Pending live)
- `/item/:type/:slug` ‚Üí channel `item-{type}-{slug}`, invalidates `episodesQueryKey(type, slug) + listsQueryKey + ["list"]`-Prefix + `["co-watchers"]` (Cross-Cutting: Partner-Ticks updaten Listen-Badges + Mitseher live) + `item_history` (`["movie-co-watchers"]`/`["movie-seen"]`) + `item_notes` (`["item-notes"]` ‚Äî Co-Member-Notizen live)
- `/calendar` ‚Üí channel `calendar`, `episode_watches` invalidiert `calendarQueryKey` + `["calendar","co-watchers"]`, `episodes` nur `calendarQueryKey`
- **BottomNav (global, mountet einmal)** ‚Üí channel `global-invitations`, `list_invitations` invalidiert `["invitations","mine"]` ‚Üí das Listen-Tab-Badge tickt von jeder Route aus live

**Cache-Fan-out f√ºr Mutations:** Jede Write die `itemCount` / Title/Watch-Beziehung √§ndert, MUSS `listsQueryKey` + `["list"]`-Prefix invalidieren. Episode-Mutations zus√§tzlich `episodesQueryKey`. Pattern in AddSheet, RowActions, ItemHeaderActions, ItemDetail toggleMut + cascadeMut.

**Anti-Pattern aus Logbook das wir NICHT machen:** Auf SUBSCRIBED ein Refresh feuern. In Logbook f√ºhrte das zu Re-Render pro Page-Mount + zerschoss den Router-Cache. Brauchen wir hier nicht, weil staleTime + Mutations + Postgres-Events alle Wege abdecken.

### Solid / Router / Reactivity
